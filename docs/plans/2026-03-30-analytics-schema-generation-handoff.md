# Analytics Schema Generation Handoff

## What generates `packages/analytics/src/schema.ts`?

`packages/analytics/src/schema.ts` in `connect-monorepo` is a generated OpenAPI TypeScript artifact.

- Generated by: `openapi-typescript`
- Source of truth OpenAPI spec: `/Users/aphex/repos/metamask/metamask-sdk-analytics-api/api.spec.yml`
- The `connect-monorepo` itself does **not** currently contain a dedicated `generate:schema` script for this file.

## Where this is defined in analytics API repo

In `/Users/aphex/repos/metamask/metamask-sdk-analytics-api/makefile`:

```make
schema:
	deno run --allow-read --allow-env --allow-write npm:openapi-typescript ./api.spec.yml -o ./integration-test/generated/schema.ts
```

In `/Users/aphex/repos/metamask/metamask-sdk-analytics-api/README.md`:

- `api.spec.yml` is documented as the single source of truth.
- `make schema` is documented as the regeneration command.

## How to regenerate directly into connect-monorepo

From anywhere on local machine:

```bash
npx openapi-typescript "/Users/aphex/repos/metamask/metamask-sdk-analytics-api/api.spec.yml" -o "/Users/aphex/repos/metamask/connect-monorepo/packages/analytics/src/schema.ts"
```

Then run formatting/lint in `connect-monorepo` to match repository style.

## Practical workflow for future updates

1. Update schema contract in analytics API repo: `api.spec.yml`
2. Regenerate schema in analytics API repo (`make schema`) to validate contract changes there
3. Regenerate `connect-monorepo/packages/analytics/src/schema.ts` from the same `api.spec.yml`
4. Run lint/tests in `connect-monorepo`
5. Open PRs in both repos if contract and client types changed together
